## 📋 Обо мне

Я QA-инженер с практическим опытом функционального, регрессионного и интеграционного тестирования веб- и мобильных приложений. Работаю в краудтестинговой компании **Testbirds** (проект **ivi**), где ежедневно взаимодействую с реальными пользовательскими сценариями и сложной инфраструктурой.

- 🔍 Провожу ручное и автоматизированное тестирование (Java / Kotlin).
- 🛠️ Работаю с API (REST, SOAP), базами данных (SQL, NoSQL) и инструментами перехвата трафика.
- 📈 Постоянно улучшаю процессы тестирования и документацию.
- 🎓 Имею профильное образование и сертификаты (см. раздел ниже).

### 1️⃣ [Автоматизация на Java](https://github.com/nalbilee/JavaTestAutomation/tree/master)

**Описание**: Разработал фреймворк для автоматизированного тестирования веб-приложения, покрывающий UI и API слои.

**Что сделано**:
- Спроектировал архитектуру тестового фреймворка с использованием **JUnit 5**, **Selenium WebDriver** и **RestAssured**.
- Реализовал паттерн **Page Object Model** для UI-тестов, что упростило поддержку и масштабирование.
- Написал API-тесты для проверки ключевых эндпоинтов (GET, POST, PUT, DELETE).
- Настроил параметризованные тесты и тестовые наборы для разных окружений.
- Интегрировал тесты в CI/CD для автоматического запуска.

**Результат**: Автоматизировал **25+ тестовых сценариев**. Сократил время регрессионного тестирования на **40%**. Выявил **8 регрессионных дефектов** на ранних этапах разработки.

### 7️⃣ [Charles Proxy](https://github.com/nalbilee/charlesproxy)

**Описание**: Использовал Charles Proxy для перехвата и модификации сетевого трафика при тестировании мобильных и веб-приложений.

**Что сделано**:
- Настроил прокси для перехвата трафика между мобильным устройством и сервером.
- Применял **Map Local** и **Map Remote** для подмены ответов сервера.
- Тестировал поведение приложения при разных HTTP-статусах (500, 404, 429).
- Использовал **Throttle** для симуляции медленного соединения (2G, 3G, Edge).
- Настраивал **Rewrite** для модификации запросов и ответов на лету.

**Результат**: Проверил **15 edge-кейсов**, которые невозможно воспроизвести стандартными методами. Сократил время на проверку обработки ошибок на **50%**.
